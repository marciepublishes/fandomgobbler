# Privacy Policy

FandomGobbler is a local-first browser extension for tracking fanfiction reading activity. This document describes what the extension does based on the current source code.

## Data Stored Locally

FandomGobbler stores extension data in the browser, mainly with `chrome.storage.local`. Stored data can include:

- tracked works and their metadata, such as title, author, URL, fandoms, relationship, summary, word count, chapter progress, stats, dates, and reading status
- custom categories, ratings, notes, author notes, hidden-rule settings, relationship groups, author watch settings, and author watch matches
- import state for AO3 bookmarks and Marked for Later
- theme, dashboard, popup, floating-button, companion, and custom cursor settings
- Google Sheets sync state if enabled, such as whether sync is enabled, the spreadsheet ID, sync status, pending deletion markers, and the signed-in account email used to detect account switches

The extension also uses a small amount of `localStorage` or `sessionStorage` for page-theme boot and temporary UI dismissal state.

## Does Reading Data Leave The Browser?

By default, your fic history, notes, ratings, bookmarks/imported works, categories, and reading status are stored locally in your browser extension storage.

That data leaves the browser only when you use features that require network access, such as:

- optional Google Sheets sync
- AO3 bookmark import
- AO3 Marked for Later import
- AO3 author watch checks
- AO3 subscription/detail refreshes
- AO3 availability checks for tracked works

The extension does not send your library data to a developer-run server.

## Optional Google Sheets Sync

Google Sheets sync is optional. If you connect it, the extension uses Google OAuth and the Google Sheets/Drive APIs to create or find a spreadsheet named `FandomGobbler Sync (Do Not Edit This File)` in your Google Drive.

The spreadsheet can contain synced work data, including titles, authors, URLs, statuses, categories, fandoms, relationships, summaries, notes, ratings, stats, series information, chapter progress, timestamps, deletion markers, and generated viewing tabs.

The extension requests Google access for:

- `https://www.googleapis.com/auth/spreadsheets`
- `https://www.googleapis.com/auth/drive.file`

The code also fetches the signed-in Google account email from Google's userinfo endpoint so the extension can detect account switches. OAuth tokens are handled by the browser identity API or stored locally for browser-specific auth flows.

Google Sheets data is stored in your Google account, subject to Google's services and your Google account settings. Disconnecting or uninstalling the extension does not automatically delete a spreadsheet already created in your Google Drive.

## Other Network Requests

The extension makes network requests to:

- Archive of Our Own (`archiveofourown.org`) for AO3 page features, bookmark import, Marked for Later import, author watch checks, subscription/detail refreshes, and availability checks
- FanFiction.net (`fanfiction.net`) through content scripts on FanFiction.net pages
- Google OAuth and Google APIs when Google Sheets sync is configured

AO3 fetches use the browser's existing AO3 session cookies (`credentials: include`) so the extension can access pages that your browser account can already access.

The extension source does not load external runtime scripts or CDN resources from extension pages. Script files used by the popup and dashboard are bundled with the extension.

FandomGobbler is an independent project and is not operated by the supported fanfiction sites, browser vendors, or Google.

## Permissions

The extension requests these permissions:

- `storage`: stores tracked works, settings, notes, categories, sync state, and related extension data locally
- `activeTab`: reads the current tab context for popup/current-work features
- `notifications`: shows extension notifications for events such as possible deleted works or author watch matches
- `identity`: supports Google OAuth for optional Google Sheets sync
- `alarms`: supports scheduled/background extension checks

The extension requests host access for:

- `archiveofourown.org`: AO3 tracking, imports, author watch, availability checks, and page enhancements
- `fanfiction.net`: FanFiction.net tracking/page enhancements
- Google OAuth/API domains: optional Google Sheets sync

## Developer Server, Tracking, And Sale Of Data

The current code does not send data to a developer-run server. The developer does not collect, sell, or track user data through a separate analytics service in this codebase.

Network requests are made to the supported fanfiction sites and to Google services when needed for the features described above.

## User Control And Deletion

Users can manage or remove tracked works, notes, ratings, categories, hidden rules, settings, and sync state through the extension UI where those controls are provided.

Users can export and import extension data using the extension's export/import features.

Uninstalling the extension removes its browser-managed local extension storage. Data that was synced to Google Sheets remains in the user's Google Drive until the user deletes that spreadsheet or removes access through their Google account settings.
