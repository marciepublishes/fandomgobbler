FandomGobbler. Progression Map

Action plan for Phases 1–3: browser expansion, product stabilization, and core/adaptor refactor.
The strongest path is to make FandomGobbler. excellent everywhere before broadening further. That means distributing first, stabilizing second, and modularizing third.

Timeline Overview

Recommended pace: about 4 months total. Fast enough to keep momentum, slow enough to preserve quality.

Phase Breakdown

4–8 weeks

Phase 1: Release Everywhere

  • Ship Chrome update at 2.0.0 quality
  • Package and publish for Edge
  • Adapt and submit for Firefox
  • Test Safari feasibility, workflow, and parity
4–6 weeks

Phase 2: Stabilize and Build Trust

  • Fix real user bugs from live usage
  • Improve backup confidence and reminders
  • Tighten docs, screenshots, and support copy
  • Add only low-risk, high-value polish
4–8 weeks

Phase 3: Refactor for Growth

  • Separate tracker core from AO3-specific code
  • Define shared item and storage models
  • Isolate AO3 extraction and injection logic
  • Prepare for future FanFiction.net/Wattpad adapters

Deliverables by Phase

Phase 1 Deliverables

  • Chrome listing updated and approved
  • Edge listing live
  • Firefox version submitted or live
  • Safari feasibility report and shipping plan

Phase 2 Deliverables

  • Post-launch bugfix release(s)
  • Polished listing copy and screenshots
  • Backup/export confidence improvements
  • Basic user help/support materials

Phase 3 Deliverables

  • Documented tracker core responsibilities
  • Documented AO3 adapter responsibilities
  • Cleaner theme/storage/render boundaries
  • Technical base for Story Tracker expansion

Guiding Rule

  • Distribute first
  • Stabilize second
  • Modularize third
  • Only then expand beyond AO3
Suggested long-term sequence after Phase 3: FanFiction.net, Wattpad, Tumblr, then broader reading targets like blogs, Substack, longform articles, news, and research reads.
